RailsCms setup guide

This guide covers the setup process of RailsCMS engine

After reading this guide, you will know:

  • How to create basic rails project with RailsCms support
  • How to extend existed project with RailsCms capabilities

1 How to setup RailsCms?

This is a simple steps, to set up RailsCms project from scratch on Linux systems.

1.1 Brief Instructions:

If you already have existed project, and you want to extend it with RailsCms, you can skip steps 1 and 2.

    1. Create new RVM (rails 3.2.*) -- strongly recommend to use RVM anywhere, anywhen.
    1. Create new Rails-project, setup and configure mysql or sqlite database.
    1. Download and install RailsCms as a local gem. To do it, just place the railscms folder in your rails root application folder, and add to Gemfile:
gem 'railscms', :path => 'railscms'
    1. Run bundle rake command.
    1. Run rake cms:wizard rake command to setup devcms. See output to understand what modifications it applied.
    1. Start the rails server, using rails s
    1. Open in your browser http://localhost:3000. You should see the admin panel. Use username admin and password admin to log in.

1.2 Detailed instruction:

If you already have existed project, and you want extend it with RailsCms capabilities, you can skip steps 1 and 2.

Assuming that you already have installed RVM. If not, visit [(https://rvm.io/)]. It is not necessary, but strongly recommend. If you don't want use it, skip step 1.

  • 1. To begin from scratch, you need to create a simple rails project. We recommended to starts with new rvm instance. Open a terminal, navigate to a folder where you have rights to create files, and type something like:
$ mkdir blog
$ cd blog
touch .rvmrc

This will create a your future project folder, and .rvmrc file. This file will be used to setup your project's ruby environment when you switch to the project root directory. Open this file with any text editor, and paste a single line (the ruby versions will be differ on your machine!):

rvm use ruby-1.9.3-p392@blog --create

The installed ruby versions can be differ on your machine. You can check your version by using rvm list terminal command.

After that, you can activate your rvm, by directory switching:

$ cd ..
$ cd blog

Choose Yes on rvm prompt. Now, you can download rails gem for your rvm version.

$ gem install rails
$ cd ..
  • 2. Now, you can create a Rails application
$ rails new blog

This will create a Rails application called Blog in a directory called blog (where we had placed .rvmrc file already). Run the bundle command, to download all required gems:

$ bundle

Setup your database

$ rake db:prepare

This will instantiate a sqlite database. If you want mysql database instead, open the Gemfile file, and replace gem 'sqlite3' with gem 'mysql2' and configure config/database.yml database configuration.

  • 3. Download and install RailsCms as a local gem. To do it, just place the railscms folder in your rails root application folder, and add to Gemfile:
gem 'railscms', :path => 'railscms'
  1. Run in terminal
$ bundle
  1. Run rake command to setup devcms. See output to understand what modifications it applied:
$ rake cms:wizard
  1. Start the rails server: bash $ rails s
  2. Open in your browser http://localhost:3000. You should see the admin panel. Use username admin and password admin to log in.
